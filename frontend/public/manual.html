1界面总览：三栏布局

编辑台采用全屏三栏式布局，从左到右依次为问题队列、当前行/时间轴、Inspector 面板。

① 左栏：Issue Queue
  • 问题统计（P0 / P1 / P2）
  • 过滤标签
  • 问题条目列表
  • Character Cast 角色列表
② 中栏：Current Line + Timeline
  • 源文 / 配音稿 双栏展示
  • 局部预览播放器
  • Original Dialogue 波形
  • Generated Dub 波形
  • 字幕单元时间轴
③ 右栏：Inspector
  • Segment Inspector
  • 配音稿编辑框
  • CLIP 信息（状态/时长/策略）
  • 批准 / 复核 操作按钮
  • Character Cast（角色信息）
  • Candidate Tournament
编辑台主界面 — 左侧 Issue Queue 显示 112 open，中央为三轨时间轴，右侧 Inspector 等待选择片段
顶部状态栏显示 Benchmark 分数（本例 69.6）和质量状态（Review Required）。 分数低于 75 或存在 P0 问题时，状态为 Review Required（橙色）；高于 85 且无 P0 时为 Good（绿色）。

2从任务详情页进入编辑台

每个已完成的流水线任务都有对应的专业配音编辑台入口。

STEP 1打开任务详情页

在侧边栏点击 任务列表，找到对应任务，进入任务详情。

任务详情页 — 底部操作栏包含「导出成品」「说话人审查」「配音返修审查」和「专业配音编辑台」四个入口按钮

STEP 2点击「专业配音编辑台」

点击蓝色的 专业配音编辑台 按钮，即可跳转至全屏编辑台页面（路径：/tasks/<id>/dubbing-editor）。

编辑台会自动从流水线输出目录中加载所有数据（字幕单元、配音稿、角色、CLIP 等），无需手动导入。

3问题队列：过滤与浏览

左侧 Issue Queue 展示所有自动检测到的配音问题，支持按严重等级过滤。

优先级含义典型类型处理建议
P0 严重缺陷，必须处理 角色音色冲突（pitch_class_drift） 立即修改配音稿或重新合成
P1 较严重，建议处理 声纹一致性失败（speaker_similarity_failed） 审听原始音频后决定是否修改
P2 轻微，可选处理 时间轴溢出（overflow_unfitted） 若不影响观感可忽略

操作点击过滤标签

点击 all open P0 P1 P2 resolved 可快速过滤问题列表。

选中 P0 过滤标签后，列表仅显示 39 条「角色音色冲突」问题，统计卡片高亮红色
建议工作流：优先处理 P0 问题（红色），再依次处理 P1、P2。处理后切换至 resolved 标签确认已完成数量。

4片段审核：选中问题查看详情

点击任意问题条目后，中栏 Current Line 展示对应片段的源文和配音稿，右栏 Inspector 显示完整详情。

操作点击问题条目

选中 P0「角色音色冲突」后 — 中栏展示源文「那得最高规格接待啊」、配音稿「This is the highest reception.」、状态 needs_review； 右栏 Inspector 显示 seg-0040、CLIP 时长 2.01s、Fit 策略 compress，及 Character Cast（SPEAKER_04，Pitch: low · 131.2Hz）

说明Inspector 各字段含义

字段含义
seg-xxxx片段 ID，与 task-c 翻译结果一一对应
CLIP 状态placed = 已安置；overflow_unfitted = 溢出未完全压缩
Fit 策略compress = 压缩速率以适配时间；overflow_unfitted = 合成后仍超时
Pitch角色音高等级（low / mid / high）及具体 Hz 值
Voice lockUnlocked = 允许自动匹配声线；Locked = 锁定指定声线
Speaker failed该角色下声纹匹配失败的片段数及占比

5局部预览：Render Range

选中片段后，可一键生成该片段前后 1 秒范围的混音预览，直接在浏览器中试听效果。

STEP 1选中一个片段（点击问题条目或时间轴）

STEP 2点击顶栏「Render Range」按钮

未选中片段时按钮呈灰色禁用状态；选中片段后按钮变为深色可点击。渲染过程约 1–3 秒。
Render Range 完成后，中栏右侧出现蓝色「局部预览」音频播放器，时间范围 02:06.46–02:10.46，时长 4 秒，可直接点击播放
预览音频包含原始背景音乐 + 当前配音片段的混合结果，可快速判断配音效果是否自然。若效果不理想，修改配音稿后可再次 Render Range 对比。

6修改配音稿文本

右栏 Inspector 的「配音稿」文本框可直接编辑，修改后点击「保存」或进行审核操作即自动保存。

操作在 Inspector 配音稿框中直接编辑

将配音稿从「This is the highest reception.」修改为「This is the highest reception standard!」—— 文本框呈蓝色边框激活状态
修改文本后点击 标记已修 或 仍需复核 按钮，系统会同时保存文本修改和审核状态，无需单独保存。
文本修改仅更新编辑台数据库中的稿件记录，不会自动重新触发 TTS 合成。如需重新合成，需在流水线中重跑 Task D。

7批准或标记复核

每个片段可设置为「已修」或「仍需复核」两种审核状态。

按钮操作后状态何时使用
✓ 标记已修（绿色） approved 配音效果满意，问题已解决
⚠ 仍需复核（黄色） needs_review 配音有问题但暂时无法处理，标记留待后续跟进
点击「标记已修」后 — Inspector 显示 approved 状态，中栏状态徽章同步更新， 左侧问题计数从「2 approved」变为「3 approved」，P0 未解决数从 39 变为 38
批准操作即时写入数据库（JSONL append-only 日志），刷新页面后状态保持不变。

8从时间轴直接选择片段

中央底部时间轴的「Subtitles / Dialogue Units」轨道上，每个片段均可点击，不必通过问题队列。

点击时间轴上的 seg-0001 — Current Line 更新为「娜娜 你在哪儿啊」→「Nana, where are you?」， Inspector 显示 SPEAKER_00、2.13s、Fit 策略 overflow_unfitted（配音略长）
时间轴片段的颜色会反映问题严重程度：橙色/红色表示有 issue，蓝色表示已批准，灰色表示未审核。

9查看已处理问题（resolved）

点击过滤标签 resolved 可查看所有已批准的问题，确认审核进度。

resolved 视图 — 显示「3 open · 3 approved」，已完成 3 条问题，划线标注表示该问题已解决

10Benchmark 质量评分说明

顶栏右侧显示当前任务的综合质量评分。

评分范围状态标签颜色建议
≥ 85 且无 P0Good● 绿色可直接导出
75–84 或有少量 P0Review Required● 橙色建议处理后再导出
< 75 或大量 P0Needs Work● 红色必须处理后导出
Benchmark 分数综合考量：声纹一致性（MOS 评估）、时间轴适配率（no overflow）、字符速率合理性。分数实时反映当前任务状态，不随审核操作自动刷新 — 重新跑 benchmark 需通过流水线触发。

11角色档案与风险标记

Inspector 底部 Character Cast 区域展示当前片段所属角色的档案信息。

SPEAKER_00 角色档案 — Pitch mid · 213.3Hz，Voice lock Unlocked，33 条片段中 6 条声纹失败（18%）， 底部 Candidate Tournament 显示「当前片段没有返修候选」
字段说明
Pitch角色音调分类（low/mid/high）及基频 Hz
Voice lockLocked = 锁定特定声线模型；Unlocked = 自动匹配
Segments该角色的总片段数
Speaker failed声纹验证未通过的片段数及百分比（橙色预警）
Candidate Tournament多候选配音片段的对比列表（如有返修候选则显示）

12导出最终成品

完成审核后，点击顶栏右上角的 Export 按钮，或返回任务详情页点击「导出成品」。

推荐工作流总结：
  1. 过滤 P0 → 逐条审听（Render Range） → 修改配音稿 → 标记已修
  2. 过滤 P1 → 重点审听声纹问题 → 处理或标记复核
  3. 切换 resolved 标签确认审核覆盖率
  4. 顶栏 Export → 下载 MP4 成品
导出操作会直接使用流水线已生成的 task-e 混音结果。若修改了配音稿文本，需重跑 Task D（语音合成）和 Task E（混音）后再导出，否则导出内容不含文本修改。
