# v0.1.1 — Video perception

> Until now translip's "perception" of a video covered audio (ASR/voiceprints) and text (OCR) only. This release adds a pair of common-sense eyes: a locally-run Qwen3-VL-4B vision-language model that answers "what's on screen, who's there, what's the mood, what text is visible".

## New features

- **Video Content Analysis atomic tool**: upload a video and pick a task —
  - *Scene description*: structured JSON per time span (location / people / action / mood);
  - *On-screen text triage*: combined with subtitle detection, classifies each text event as dialogue subtitle, scene text (signs/storefronts), watermark, or title card;
  - *Erase QC*: checks the erased video for leftover text or inpainting artifacts;
  - *Free-form Q&A*: ask anything about the video ("What car appears?" → "A white Mercedes convertible").
- **`asr-dub+visual` workflow template**: inserts a visual-perception node into the basic dubbing chain, injecting per-span scene descriptions (e.g. "inside a car, a man and a woman talking, tense mood") into translation context. On the same test clip, 20 of 24 lines improved — the opening line went from the literal "Paradise Dubai, Lao Jia" to "Old Jia, living the high life in Dubai!".
- **`translip analyze-video` CLI**: the same engine behind the tool and the pipeline node, scriptable from the command line.

## Engineering & stability

- **Fully local, optional install**: Apple Silicon uses MLX (4-bit, ~3.3 GB, auto-downloaded on first use); other platforms can point at a local Ollama with zero extra dependencies. Skipping the vision extra leaves everything else untouched.
- **Graceful degradation**: the visual node is optional — a missing backend or a failed analysis never fails translation or the pipeline.
- **Heavy-tool serialization**: multi-GB-model tools (vision, subtitle erase, separation) now queue on a single slot, so a 16 GB machine can't be exhausted by two large models at once.
- **Cache-aware**: visual analysis results are fully cached; changing only translation parameters skips re-analysis, while changing frame sampling recomputes exactly the visual and translation nodes.

## Performance reference (M4 / 16 GB, measured)

- Warm model load ~3.4s; single-frame inference 2.5–3.3s; 4 frames ~8s. A 90-minute film takes roughly 40 minutes for full scene perception (cached for reuse).
