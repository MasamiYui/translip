# v0.0.9 — Atomic tools / Works library

> Two themes this round: **break apart** and **bring together**. Break the pipeline into small, individually-callable tools; bring multi-task, multi-clip footage together under the Works library.

## New features

- **Atomic toolset**: the Tools page exposes single-step tools (separation, ASR, speaker clustering, dubbing, mixing, subtitle detection, …) so you don't have to run a full pipeline template. Each invocation gets its own Job detail to trace inputs/outputs.
- **Works & Character library**: Works can attach TMDb metadata and reuse posters/cast; Characters remember voiceprints and preview clips, so new tasks can reuse existing characters instead of assigning ad-hoc.
- **Pipeline visualization**: task detail now has a "Pipeline graph" tab showing node dependencies, status, duration and cache hits.

## Improvements

- OCR subtitle detection graduates from "experimental" to a first-class option, enabled via the `+OCR` and `+erase` templates.
- The delivery panel on the task detail page has been rebuilt to clearly separate preview from final, making side-by-side comparison easier.

## Fixes

- Fixed an occasional cache-key collision when running tasks in parallel.
- Fixed timestamp drift in the ASR stage on certain Japanese sources.
