# v0.1.0 — Dubbing Workbench & pipeline stability

> This is the first release we tag as `stable`. There are two themes: opening the Dubbing Workbench from internal preview to everyone, and consolidating default pipeline parameters and observability under Global Settings. Details below, broken down by area.

## New features

- **Dubbing Workbench is GA**: enter it from the task detail page to preview voices per character, swap timbres, fine-tune durations, and push every edit back into the task draft in one click. See [docs/dubbing-editor-manual.html](../../docs/dubbing-editor-manual.html).
- **Changelog hub**: Global Settings now has a "Release notes" group; every notable change ships here and you can open each release's detail page — exactly the page you are reading right now.
- **One-click missing-model download**: Settings → Model status now exposes a "Download missing" button that batches every missing weight, with cancel, per-item status and retry on failure.

## Improvements

- **Global task defaults**: separation, transcription, translation, dubbing, mixing, subtitles and subtitle-erase defaults all live in Global Settings → Task Defaults. New tasks pick them up automatically — no need to fill the same fields every time.
- **Bilingual subtitle layout, rebuilt**: bilingual subtitles can now place the Chinese and English tracks independently; preview rendering and final delivery stay in sync.
- **Window ducking in the mixer**: two new knobs — `window_ducking_db` and `max_compress_ratio` — let you keep ambient layers alive while making voices pop.

## Fixes

- Fixed an occasional timeline offset in the time-stretch stage when source footage used non-integer frame rates.
- Fixed model downloads stalling on lossy networks; downloads now auto-reconnect and resume.
- Fixed an edge case in OCR subtitle erase where small mask dilation left visible residue.

## Upgrade notes

- The global config file structure has changed; the first launch will migrate any existing config automatically.
- If you used to control `dub_repair_*` via environment variables, those still work but are now superseded by the "Dub repair" panel in Global Settings.

Older release notes will keep landing in the "Release notes" group. If anything broke for you after upgrading, please reach out via Global Settings → About.
