# 上云配音,最贵的从来不是合成——国产云端 TTS 后端的一次价格普查

> 这是 **幕后 · 博客** 的第十篇,也是第四篇**决策记录**。上一篇 [《排行榜会在 Mac 上骗你》](/blog/tts-apple-silicon) 的结尾,我留了个没填的坑:跨语种音色那顶皇冠——用中文演员的音色去念英文、还带着戏——**本地 Mac 暂时拿不到**,诚实的路只有两条,要么上一台带 CUDA 的远程机器,要么**加一个 opt-in 的 API 后端**,像 [本地优先](/blog/local-first) 那篇立的规矩:"云是一个你主动打开的开关。"我当时把开关画在了图上,却一个字没说**这开关后面接哪朵云、按一下要花多少钱**。这一篇,把这道题答完。(上一篇说好下一篇回算法线讲 Task E,那一棒我先欠一周——价目表这事儿趁热,凉了就又改版了。)

## 先把要回答的问题摆正

我们要的不是"上传视频、吐出配好音的成片"那种端到端黑盒。translip 的配音是 [一条分了站的流水线](/blog/pipeline-orchestration):人声背景分离、说话人归属转写、翻译、**TTS 合成**、时间轴贴合与混音。我们只想把其中 **task-d 这一站**——"文本 + 目标音色 → 一条干声"——换成云端,**翻译、对口型、混音全留在本地自己手里**。

所以这次普查,盯的是六件很具体的事,排序就是影视剧配音的优先级:

1. **接得进吗** —— 有没有能塞进 task-d 的 REST/流式 API,时长语速能不能精确控;
2. **像不像** —— 音色克隆要喂多少声音、还原度口碑、能不能跨语种克隆;
3. **有没有戏** —— 情感、语气、停顿这些影视剧的魂;
4. **跨不跨得了语种** —— 中↔英及更多语种的可用音色;
5. **多少钱** —— 按字符/秒的真实单价、克隆是否另收费、免费额度;
6. **合不合规** —— 复刻真人声音的实名与授权门槛。

国内重点挖了 **火山引擎(豆包)、阿里云百炼(CosyVoice / Qwen-TTS)、腾讯云、科大讯飞、百度智能云、MiniMax(海螺)** 六家,补充对比 **Fish Audio、硅基流动 SiliconFlow** 两个绕不开的角色,再拿 **ElevenLabs** 当那把"质量天花板"的尺子。

> 说在前头:这篇的价格全部来自 2026 年 6–7 月的公开检索 + 多源交叉核验,**不是凭记忆写的**。但你马上会看到,**相当一部分厂商的真实分档单价,根本不在能抓到的公开页面上**——它藏在登录后的控制台里。这本身就是这次调研最大的一条发现,我会在专门一节里把它摊开。

## 第一道关:这些云,真能插进 task-d 吗

选型最容易被忽略、却最先卡住人的,不是音色,是**接入形态**。我把五家国内厂商的合成接口排了一遍,结论有点出人意料:

- **MiniMax(海螺)** 是唯一一个提供**同步 REST HTTP** 的——`POST /v1/t2a_v2`,单请求最多一万字符,非流式直接返回一段 JSON、流式走 SSE,输出 MP3/PCM/FLAC/WAV、采样率到 32 kHz。这是五家里**唯一"即插即用"的纯 REST 后端**(中国区 `api.minimaxi.com`、全球区 `api.minimax.io`,别用那个残留在三方库里的 `.chat` 老域名)。
- **火山豆包、阿里百炼 CosyVoice、腾讯云、讯飞** 全是 **WebSocket 双向流式**。它们为"大模型逐字吐字、边合成边播"设计,延迟低(豆包端到端约 750 ms),但代价是:你得在 task-d 里实现一个 WS 客户端,而不是甩一个 HTTP 请求就完事。对一个本来就异步跑、不追求实时播放的离线配音流水线来说,**WS 是纯增加的工程量**。

![五家厂商的接入形态:只有一家是即插即用的 REST](/blog/cloud-tts-dubbing/fig-api-reality.svg)
*把"文本+音色→语音"这一步接进 task-d:MiniMax 是唯一的同步 REST,其余四家都是为实时流设计的 WebSocket。而五家**无一例外**只给"倍速缩放"(0.5–2.5×)的语速控制,没有一家给"逐句目标时长锁定"——这恰恰说明 translip 自己的时间轴贴合,不是可选项。*

还有一条横跨五家的共性,对配音对口型至关重要:**没有任何一家提供"逐句目标时长(duration)精确锁定"的参数。** 它们给的都是"倍速缩放"——MiniMax 的 `speed` 0.5–2.0×、CosyVoice 的 `rate` [0.5, 2]、腾讯的 `Speed` [-2, 6] 映射 0.6–2.5×。CosyVoice 的文档甚至直说"大模型当前不支持时间戳",官方的对口型变通法就是:**按 `目标时长 / 实际时长` 反复缩放 `rate`**,凑近为止。

这意味着什么?**意味着你换不换云,[Task E 的时间轴贴合(atempo / rubberband)都得留着。](/blog/pipeline-orchestration)** 云端给你一句长短不一的干声,把它精确塞回原片那一格格的时间线、再二次拉伸到不破音——这活儿云不替你干。选型不改变这条架构的必要性,反而印证了它。

## 第二道关:音色克隆,要喂多少声音、像不像

这是影视剧配音的命根子。挖下来,克隆能力最能打的是这么几家:

- **MiniMax** 证据最硬。它用一个**可学习的说话人编码器**,从参考音频里直接抽音色、**不需要那段音频的转写文本**(论文 [arXiv:2505.07916](https://arxiv.org/pdf/2505.07916) 原文:"extracts timbre features from a reference audio without requiring its transcription"),是真正的本征零样本克隆。架构上几秒就够,Speech 2.5 的快克隆压到约 6 秒;但**生产环境的 02 代克隆 API 实际要求音频 10 秒–5 分钟**(≤20 MB),约 30 秒铸出一个可复用的 `voice_id`。官方自报约 99% 相似度——记住这是厂商口径,不是第三方实测。
- **Qwen3-TTS** 是另一极:**Apache-2.0 真开源**(0.6B / 1.7B 权重可下载),约 **3 秒**参考音频就能克隆,覆盖 10 语种 + 北京/四川方言,**单个克隆音色能说任意支持语种**(跨语种克隆),它自报的跨语种说话人相似度 0.789 甚至高于 MiniMax 与 ElevenLabs。注意区分:GitHub 上那个开源自托管的 Qwen3-TTS,和阿里云百炼**托管的** Qwen-TTS API,是两个不同的东西——translip 的 task-d 里其实[已经有一个 `qwen3tts` 后端的引子](/blog/voice-cloning)。
- **阿里云 CosyVoice** 的克隆有个别人没有的好处:**创建复刻音色本身免费**,每个账号最多攒 1000 个音色,复刻音色的合成和公版音色同价。坏处是超过 1 年没用会被自动下线、还不支持手动删。

样本时长这一栏值得单拎出来对比——它直接决定"拿一段戏里的台词能不能就地克隆":MiniMax 生产 API 约 10 秒起、Qwen3-TTS 约 3 秒、ElevenLabs 的即时克隆(IVC)≥1 分钟、专业克隆(PVC)要 30 分钟到 3 小时的棚录。对"从原片里截一段角色声音就想复刻"的影视场景,**越短越友好**,这一项 Qwen3-TTS 和 MiniMax 明显占优。

## 第三道关:情感与跨语种——影视剧的魂

干巴巴地把字念对,不叫配音。影视剧要的是语气、停顿、哭腔、那口气。

- **火山豆包** 在情感标签上铺得最开:原生 **28 种情感/风格**(`emotion` 标签,如 `excited`),还能 `emotion_scale` 0–5 调强度。但有个门槛——**情感受音色门控,只有带 `_emo_` 的音色才能用**(比如 `zh_female_..._emo_v2_...`),省略 `emotion` 时它自动预测。
- **MiniMax** 给的是**离散情感枚举**(happy / sad / angry / fearful / disgusted / surprised / neutral)外加 `auto` 让模型按文本自动判断,配合 `pitch`(±12 半音)、`speed`,以及文本里 `<#x#>` 的停顿标记(0.01–99.99 秒)。

跨语种这顶皇冠,正是上一篇说"本地拿不到"的那顶。云上谁戴得稳?**MiniMax** 支持 32→40 语种、同一段话里能切语种,还号称跨语种切换时**保留目标音色的口音口癖**。但这里要给一个诚实的对抗核验结果:网上盛传的那条"MiniMax 跨语种 WER 全面优于 ElevenLabs、在中粤泰越等声调语言显著胜出",在我们的三票核验里**被 3:0 否决了**——它来自论文自述、不可作为独立结论。更说明问题的是 MiniMax 自己的版本节奏:

| MiniMax 版本 | 发布 | 关键升级 | 克隆样本 |
| --- | --- | --- | --- |
| Speech 02 | 2025-05 | 32 语种、本征零样本克隆 | ~10 秒(API) |
| Speech 2.5 | 2025-08 | 40 语种、英文相似度提升、跨语种**保留**口音 | ~6 秒 |
| Speech 2.6 | 2026-03 | 端到端 <250 ms、Fluent LoRA | ~10 秒 |
| **Speech 2.8** | **2026-06-05** | **专项修复跨语种"口音渗漏"(accent bleed)** | **~5 秒** |

看到没——**2.8 是在我们做这次调研的当天发布的**,主打的恰恰是"消除口音渗漏、让每种语言听起来像母语者"。这反过来证明:**2.5 那个"保留口音"在严肃跨语种配音里其实是个 bug 不是 feature**(中文音色念英文,你要的是地道英文、不是中式口音英文)。这顶皇冠还在被一版一版地打磨,**别把任何一版的厂商自述当终点。**

还有一条彩蛋,直接接上一篇的悬念:上一篇说跨语种最强的开源候选是 **IndexTTS 2.5 / CosyVoice 3**,但它们"**没有 Apple Silicon 路径**"。而这次发现——**硅基流动 SiliconFlow 把 IndexTTS-2 托管成了按量 API**,约 ¥1.56/万汉字,而且这一支**自带精确时长控制**。也就是说,上一篇里那顶"本地够不着"的皇冠,如今能以一个**人民币结算、OpenAI 兼容的 REST 接口**摸到了。这是整篇调研里我自己最意外的一条。

几个一眼就能读出来的形状:

- **接入最省事的是 MiniMax、SiliconFlow、Fish**(纯 REST),四家国内大厂全是 WebSocket;
- **克隆计价的差异是数量级的**:免费(CosyVoice / SiliconFlow / Fish)→ ¥9.9(MiniMax)→ 槽位几十元(火山)→ **基础版 3800 元/个**(腾讯)→ **2 万元/年·个**(讯飞在线特色音);
- **合成单价最低的是 SiliconFlow 托管的 CosyVoice2(~¥1.56/万汉字)**,最高的是 ElevenLabs(~¥11–22),中间差了近一个数量级。

## 算一笔账:一部电影,和一季剧集

光看单价没用,得落到量级上。两个场景,假设都写明:

- **场景 A · 单片电影**:约 2 万字对白、**15 个角色音色**;
- **场景 B · 一季剧集**:20 集 × 约 8000 字 ≈ **16 万字对白**、**25 个角色音色**。

把"合成费 + 克隆费"分开算,几家可嵌入 task-d 的厂商对比如下(纯中文、用上面那张表的折算口径;标 ⚠️ 的是分档未公开、按已知档保守估):

| 厂商 | 合成费 A / B | 克隆费 A / B | **合计 A(电影)** | **合计 B(剧集)** |
| --- | --- | --- | --- | --- |
| **SiliconFlow · CosyVoice2** | ¥3 / ¥25 | 免费 | **≈ ¥3** | **≈ ¥25** |
| **阿里 · CosyVoice** | ¥8 / ¥64 | 免费 | **≈ ¥8** | **≈ ¥64** |
| **Fish Audio · s2-pro** | ¥7 / ¥53 | 免费(含套餐) | **≈ ¥7** + 订阅 | **≈ ¥53** + 订阅 |
| **MiniMax 海螺(中国区)** | ¥20 / ¥160 | ¥149 / ¥248 | **≈ ¥170** | **≈ ¥410** |
| **火山豆包** | ¥16 / ¥128 | ¥1320 / ¥2200 | **≈ ¥1340** | **≈ ¥2330** |
| **腾讯云(基础版克隆)** | ¥6 / ¥48 | ¥5.7 万 / ¥9.5 万 | **≈ ¥5.7 万** ⚠️ | **≈ ¥9.5 万** ⚠️ |

这张表把那个反直觉的结论砸得很实:

> **在影视剧这个量级上,合成字符费根本不是大头。** 一部电影 2 万字的对白,用 CosyVoice 合成只要 8 块钱,用 SiliconFlow 上的 CosyVoice2 只要 3 块。哪怕最贵的 ElevenLabs,2 万字也就装进它 Creator 档(¥160/月)里绰绰有余。真正决定你账单是三位数还是六位数的,**是"克隆音色怎么计价"**——免费 vs ¥9.9/个 vs 槽位上千 vs 腾讯基础版 3800 元/个。25 个角色,光克隆这一项,CosyVoice 是 0 元,腾讯基础版能给你干到 9.5 万。

![两种量级的账单:决定金额的不是合成,是克隆怎么计价](/blog/cloud-tts-dubbing/fig-money.svg)
*上半:合成单价(元/万汉字),SiliconFlow 的 CosyVoice2 到 ElevenLabs 差了近一个数量级。下半:一部电影 / 一季剧集的总账单——彩色那截"合成费"在所有方案里都被压成一条细线,真正撑起柱子的是灰色那截"克隆费"。选型选的不是"哪家合成便宜",是"哪家克隆计价方式扛得住多角色"。*

所以成本维度的真结论是:**CosyVoice(阿里自营或 SiliconFlow 托管)在"克隆免费 + 字符极廉"上是双优**;MiniMax 贵一档,但 ¥9.9 的克隆 + 唯一的 REST + 情感 + 40 语种 + 当天发布的 2.8 口音修复,**贵得有买点**;火山的"音色槽位"和腾讯的"按版本档高单价克隆",对**多角色影视剧并不友好**——它们是给"少数几个固定主播音色"设计的计价,不是给"一部剧二十几号人"设计的。

## 价目表的下半张,藏在登录页后面

该说那条最不体面、却最该说的发现了。这次普查,**国内大厂相当一部分真实分档单价,我们在所有能抓到的公开页面上都没找到**:

- **火山豆包**:大模型语音合成的"元/万字符"后付费刊例,官方计费页被网络策略挡住,第三方汇总给到 ~5 元/万字符、声音复刻音色 ~8 元/万字符,但和某促销快照(¥22.5/10 万字符)对不齐,**得登录控制台才能定准**;
- **腾讯云**:基础/精品音色(0.2/0.3 元/万字符)是公开的,但那是**非大模型**的老音色;真正能跟 CosyVoice/MiniMax 比质量的**大模型音色、超自然音色,单价一个数字都没公开**;
- **MiniMax 中国区**:确认了"1 汉字 = 2 字符"的计费规则、确认了 T2A 由 10 元降到 5 元/万字符,但 **speech-02-hd/turbo 的中国区精确分档,文档域名被挡、抓不到**;
- **阿里 Qwen-TTS / 百炼托管**:确认按 token 计费(每秒音频 25 token),但**具体 token 单价没在公开摘要里出现**;
- **百度、讯飞**的声音复刻**正式商用价**(非邀测、非促销)同样**查无明文**。

还有一个更隐蔽的坑,会让"横向比价"变成关公战秦琼:**各家的"字符"压根不是一回事。** 阿里 CosyVoice 和 MiniMax 明确"**1 个汉字算 2 个字符**"(所以 2 元/万字符的 CosyVoice,纯中文实际是 4 元/万汉字);SiliconFlow 和 Fish 按 **UTF-8 字节**计(1 个汉字 = 3 字节,对中文是 3 倍放大);ElevenLabs 按 credit、Flash 模式还折半。**同样标"X 元/万字符",落到纯中文台词上能差出两三倍。** 我上面那张折算表已经尽量统一到"元/万汉字",但凡标了 ⚠️ 或"未公开"的格子,**请务必登录控制台按你自己的文本实测一遍再签合同。**

这条发现本身就值一条选型原则:**报价透明度,是选型的一个维度。** 一个把真实单价摊在公开页面上的厂商(MiniMax 海外区、SiliconFlow、ElevenLabs),和一个"具体价格请登录控制台/联系商务"的厂商,在"能不能提前把成本算清楚"这件事上,体验是两码事。

## 还有一道闸门:实名与授权,影视剧绕不过去

价格之外,影视剧配音还有一道**法律前置闸门**,而且是硬的——你要复刻的是**真人演员的声音**。

- **实名是底线**:所有厂商都要求**调用方账号实名认证**才能用声音复刻,MiniMax 的 `voice_clone` 明确"仅支持个人/企业认证用户调用"。
- **被复刻人的授权,责任普遍下放给你**:绝大多数平台**不在技术上强制**核验"被复刻人是否同意",而是在协议里把责任甩给开发者——"你需自行确保已获得被复刻对象的合法授权,违规后果自负"。**唯一的例外是讯飞**,它的语音复刻**强制要求在训练录音开头朗读一句指定授权语句**,不读就训练失败——这是把合规做进了流程。
- **法规层面是刚性的**:《互联网信息服务深度合成管理规定》白纸黑字要求,提供人声编辑功能的,"**应当提示使用者依法告知被编辑的个人,并取得其单独同意**";向公众提供生成式服务前还要**履行算法备案**;生成内容要加 **AI 生成标识**。
- **数据留存**:多数厂商承诺音色存于境内(北京),MiniMax 称其接口无状态、不存输入,原则上不拿你的数据训练(除非你书面同意)。

对影视制作来说,这意味着:**复刻任何一位演员的声音之前,必须拿到书面授权并留档**。而这,恰恰又是一条支持"把云端克隆做成 opt-in 开关"的理由——它不该是默认就开着、随手就把一段声音传上云的东西,它该是一个**你清楚知道自己按下了、且手里攥着授权书才按**的开关。这和 [本地优先](/blog/local-first) 那篇的态度,严丝合缝。

## 选型:给 task-d 接哪朵云

把六道关叠起来,不存在"一家通吃",但能给出清清楚楚的**分场景推荐**:

![给 task-d 接哪朵云:按场景分的四条路](/blog/cloud-tts-dubbing/fig-selection.svg)
*没有银弹,但有四条清楚的路:省事首选 MiniMax(唯一 REST + 情感 + 40 语种 + ¥9.9 克隆 + 当天的 2.8 口音修复);批量压成本走 SiliconFlow 托管的 CosyVoice2 / IndexTTS-2(人民币、OpenAI 兼容 REST、克隆免费、IndexTTS-2 还自带时长控制);已在阿里生态里就直接用百炼 CosyVoice(克隆免费、4 元/万汉字,认 WebSocket);只有出海英文内容,才值得为 ElevenLabs 扛境外支付和访问摩擦。*

- **想最省心地先接通一个云后端 → MiniMax 海螺。** 它是唯一的同步 REST,克隆 ¥9.9、样本 10 秒起,情感枚举 + 40 语种,而且 [task-d 的 `reference_audio + ref_text → synthesize` 接口](/blog/voice-cloning)和它严丝合缝。当天发布的 Speech 2.8 主攻跨语种口音渗漏,正中你最硬的那个需求。代价是中国区合成单价偏贵(~10 元/万汉字)、精确分档要登录确认。
- **要把剧集批量成本压到地板 → SiliconFlow 托管的 CosyVoice2 / IndexTTS-2。** 人民币结算、OpenAI 兼容的纯 REST、上传参考音克隆**免费**、CosyVoice2 约 ¥1.56/万汉字,而 **IndexTTS-2 这一支还自带精确时长控制**——这是上一篇那顶"本地够不着的跨语种皇冠",如今最便宜的摸法。
- **已经在阿里云生态里 → 直接上百炼 CosyVoice。** 创建复刻音色免费、4 元/万汉字,质量是开源 SOTA 这一档;只是合成走 WebSocket,得多写个客户端,且没有逐句时长锁定(继续靠 Task E 兜底)。
- **只有出海英文内容才碰 ElevenLabs。** 它是英文和情感的天花板,但**中文明显弱于 Fish Audio**,叠加境外支付 + 国内访问受限,纯国内影视场景性价比最差。
- **火山豆包和腾讯云**:能力都在线(火山 28 情感很诱人),但**克隆计价对多角色影视剧不友好**(槽位 / 高单价档),更适合"少数固定音色 + 实时流式播报"的场景,而不是一部剧二十几号人的批量克隆。

落到 translip 的工程上,这件事的形状和上一篇说的一样轻:你那套 `TTSBackend` 协议,**加一个云后端就是一个文件 + 一行注册**,标上 `requires_network=True`,和 `deepseek` 翻译后端一个性质——平时不碍着本地默认那台 MOSS-Nano,要更像、要跨语种了,**你显式把这个开关扳过去**。接进来之后,[现成的三轴自检(ECAPA / backread / 时长)+ dub_qa 仪器](/blog/dub-evaluation)一行都不用改,直接拿来给"MOSS 本地 vs MiniMax 云端 vs CosyVoice"打分、亲耳听。

## 几条"为什么这么定"

把这次普查的取舍收拢成几条:

- **接入形态是第一道筛子,不是音色。** 一个音色再好、只给 WebSocket 的后端,接进一条离线流水线的成本,可能比音色差一点、但给 REST 的后端高得多。MiniMax / SiliconFlow 能排在前面,一半是音色,一半是那个"甩个 HTTP 就完事"。
- **比单价之前,先统一口径。** "元/万字符"在汉字×2、UTF-8 字节、credit 折半这些规则下,落到中文台词上能差两三倍。不折算到"元/万汉字",所有横向比价都是假的。
- **影视剧的账,克隆计价比合成单价重要一个数量级。** 合成费是个位数到几百元,克隆费能从 0 飙到六位数。选型要盯的是"克隆音色怎么计价扛不扛得住多角色",不是"哪家合成便宜一分钱"。
- **报价透明度本身是个选型维度。** 真实分档藏在登录页后面的厂商,让你没法提前把成本算死;能在公开页面把单价摊开的,值这一分。
- **云端克隆是 opt-in 的开关,不只因为本地优先,更因为合规。** 复刻真人声音要书面授权 + 实名 + 算法备案 + AI 标识——这东西本就不该默认开着、随手上传,它该是你攥着授权书才按下的那一下。
- **别把任何一版厂商自述当终点。** Speech 2.8 在我们调研当天发布、专修上一版"保留口音"的毛病——这赛道一个月一个版本,99% 相似度、SOTA、登顶榜单这些话,当方向看,别当判决。

## 小结

跨语种音色那顶皇冠,本地 Mac 暂时戴不上,只能 opt-in 一朵云;而这次把云挨家挖了一遍,得到一个反直觉的账本:**在影视剧这个量级,合成字符费只是个位数到几百元的零头,真正撑起账单的是"克隆音色怎么计价"和那道实名/授权的合规闸门。** 想最省心,接 MiniMax(唯一 REST + 情感 + 40 语种 + ¥9.9 克隆 + 当天的 2.8 口音修复);想把批量成本压到地板,走 SiliconFlow 托管的 CosyVoice2 / IndexTTS-2(人民币、REST、克隆免费、还自带时长控制);已在阿里就用百炼 CosyVoice(克隆免费、4 元/万汉字)。而价目表的下半张基本都藏在登录页后面,签合同前,务必用你自己的台词在控制台实测一遍。

下一篇,我**真的**回算法线,把这条流水线的最后一棒讲完——**Task E:时间线贴合与混音**:不管这条干声是本地配的还是云上配的,它长短不齐、还是一条裸声,怎么精确塞回原片那一格格的时间线、再和背景重新混成一条能交付的声音。这一篇反复说"云不替你对口型",下一篇就讲我们自己怎么对。
