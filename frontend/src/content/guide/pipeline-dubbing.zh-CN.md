# 完整配音流水线（端到端）

这是 translip 最核心的用法：一条命令式的端到端配音。界面里走六步，每一步都有专门的页面接住你。

![一条配音成片的完整旅程](/guide/flow-dubbing-journey.svg)

*图 1：从新建任务到导出成片的六个步骤；第 3–5 步可按需跳过。*

> 💡 第 4 步「配音编辑台」和第 5 步「评测与修复」内容较多，各有独立章节：[配音编辑台](/guide/dubbing-editor) · [配音评测](/guide/evaluation)。本章把主干串起来。

## 第 1 步 · 新建任务向导

点右上角「**新建任务**」进入分步向导。配音模式下是四步：**素材与语言 → 成品目标 → 质量与设置 → 确认创建**。

![新建流水线任务向导](/guide/new-task.png)

*图 2：新建任务向导的第一步——选素材、设语言，顶部是分步指示。*

### ① 素材与语言

| 字段 | 说明 |
| --- | --- |
| **任务名称** | 显示用的名字，留空会自动生成。 |
| **输入视频** | 源视频的绝对路径；点选择按钮可调系统文件框。填好后会显示时长 / 格式等媒体信息。 |
| **源语言 / 目标语言** | 原片语言与要配成的语言，决定 ASR、翻译与 TTS 的语言。 |
| **应用预设** | 载入之前保存的配置，一次性套用语言与各项参数。 |

### ② 成品目标

按你最终想要什么来选——向导会据此**自动点亮合适的阶段与模板**，你不必手动拼装：

| 成品目标 | 产出 | 典型场景 |
| --- | --- | --- |
| **配音成片**（dub_final） | 正式配音母带，不烧字幕，保留配音 + 背景双轨 | 直接发布的成片 |
| **双语审片版**（bilingual_review） | 带 OCR 字幕、保留原片画面与背景，便于对照 | 内部审片、双语校对 |
| **字幕版**（english_subtitle） | 干净底板 + 烧录字幕，OCR 优先 | 只要字幕、不要配音 |
| **快速验证版**（fast_validation） | 预览混音优先、仅 ASR、最精简链路 | 先快速看效果 |

### ③ 质量与设置

先选一个**质量档位**，再按需微调：

| 质量档位 | 取舍 |
| --- | --- |
| **快速** | 出草稿最快，预览混音，变速更保守。 |
| **标准** | 速度与质量平衡，**默认**。 |
| **高质量** | 增强混音、更精细的时长贴合。 |

这一步还能调整这些常用项（不确定就保持默认）：

| 设置 | 作用 |
| --- | --- |
| **台词校正** | 用 OCR 字幕校正 / 替换听写文本（需模板支持），可选保守 / 标准 / 激进。 |
| **TTS 后端** | 合成配音的引擎，默认 `moss-tts-nano-onnx`。 |
| **翻译后端** | `local-m2m100`（离线）或 `deepseek`（在线，需 API Key）。 |
| **ASR / 说话人分离** | 听写后端（faster-whisper / FunASR）与说话人分离开关。 |
| **设备 / 并发数** | 计算设备（auto / cpu / cuda / mps）与配音并发。 |
| **使用缓存** | 输入未变时复用各阶段产物（建议开启）。 |

> 💡 想完全手动控制？展开「**开发者设置**」可直接指定工作流模板、字幕输入策略、起止阶段、交付底板与音轨。一般用户用不到。

### ④ 确认创建

核对摘要无误后，点「**创建任务**」。这里还可以：勾「**自动绑定作品**」把任务关联到[作品库](/guide/libraries)里的剧集，勾「**保存为预设**」把当前配置存下来下次复用。

## 第 2 步 · 运行与监控

创建后自动进入**任务详情页**。这里是你跑任务时停留最久的地方。

![任务详情页：进度、阶段图、交付与产物](/guide/task-detail.png)

*图 3：顶部是总进度与状态，中间是实时阶段 DAG，下方是交付进度与成品素材清单。*

- **阶段 DAG**：实时显示每个阶段的状态与百分比；点任意节点可展开该阶段的**日志**与**产物**（音频可直接试听、JSON 可格式化查看）。状态颜色与含义见[核心概念](/guide/concepts)。
- **重跑控制**：选一个起始阶段「**从所选阶段重跑**」，之前的阶段会复用缓存；也能「停止任务」或「删除任务」。
- **台词校正报告**（若启用）：汇总「已校正 / 保留 ASR / 建议复核 / OCR 漏配」的段数。

任务跑到 100% 后，可以直接交付，也可以先进编辑台与评测精修。

## 第 3 步 · 说话人核对

在交付前，建议先核对一遍说话人——分离 / 归因难免有误判，错误的说话人会直接影响配音音色。

从详情页底部「交付」区的「**说话人核对**」进入。在这里你可以逐段确认：保留当前说话人、改判到另一个说话人、与某个说话人合并，或标记某段**不可克隆**。系统也会用风险标记（如样本过短、单段说话人、边界存疑）帮你聚焦最该看的段落。应用决策后，相关阶段会按需重跑。

## 第 4 步 · 精修配音（编辑台 + 评测）

这两步是「精修闭环」，按需使用：

- **[配音编辑台](/guide/dubbing-editor)**：以问题队列驱动，逐段试听、改译文、调时长、换音色、单段重新合成，改动一键回写任务草稿。
- **[配音评测](/guide/evaluation)**：对整条配音逐段质检打分，定位漏配 / 节奏 / 音色 / 翻译问题，并能一键自动修复。

只想快速出片，可以跳过这两步，直接进入交付。

## 第 5 步 · 交付导出

回到详情页底部「交付」区，点导出按钮打开**导出向导**（四步）：

1. **选导出配置**——按成品类型挑一个：

   | 配置 | 画面 | 音轨 | 字幕 |
   | --- | --- | --- | --- |
   | **配音无字幕** | 原片 | 配音 + 背景 | 无 |
   | **双语审片版** | 原片 | 配音 + 背景 | 双语 |
   | **字幕烧录版** | 干净底板（如可用） | 配音 + 背景 | 烧录 |
   | **仅预览** | 原片 | 预览混音 | 无 |

2. **确认素材来源**——选画面底板（原片 / 干净底板）、音轨、字幕来源（OCR / ASR）。若原片自带硬字幕，向导会提示并给出「保留硬字幕 + 叠加译文」等策略。
3. **字幕样式**（需要字幕时）——字体、字号、位置、颜色、描边、加粗等。
4. **预览并导出**——可先生成一小段**字幕预览**确认样式，再「导出成品」。导出的文件会列在「最近导出结果」里，支持内嵌播放与下载。

## 成品素材清单

详情页的「成品素材清单」随任务进度逐项点亮，方便单独取用中间产物：

| 素材 | 何时可用 |
| --- | --- |
| 原始视频 | 始终（即输入） |
| 干净画面 | 跑了字幕擦除时 |
| 纯配音音轨 / 预览混音音轨 | 跑了合成 / 回贴时 |
| OCR 英文字幕 / ASR 英文字幕 | 对应字幕阶段完成时 |
