# 核心概念：阶段、模板与缓存

这一章不写代码，只讲三件让你「用得明白」的事：流水线由哪些**阶段**组成、**模板**决定跑哪些阶段、**缓存**为什么让改一个设置后重跑还那么快。理解了它们，你就知道改完设置会重算什么、为什么能从中间某一步重跑、为什么有的阶段几秒就过。

## 流水线 = 一串独立阶段

一条配音任务被拆成若干**阶段**，按依赖顺序依次执行。每个阶段都是一个**独立子进程**：读入上一阶段落盘的文件，产出自己的文件 + 一份 `manifest`。这样设计有两个直接好处——重型模型用完即放、单个阶段崩溃也不会拖垮整条任务。

![配音流水线的七个阶段与可选支线](/guide/flow-pipeline-dag.svg)

*图 1：七个核心阶段顺序执行；OCR、画面感知、字幕擦除是按模板点亮的可选支线。*

七个核心阶段各自负责什么、用什么后端：

| # | 阶段 | 做什么 | 可选后端 |
| --- | --- | --- | --- |
| 1 | **分离** | 把人声与背景分轨 | demucs（音乐向）/ cdx23（对白向） |
| 2 | **转写** | 听写成文本并归因到说话人 | ASR：faster-whisper / FunASR；分离：ECAPA / pyannote |
| 3 | **说话人注册** | 为每个说话人建立音色档案 | 跨任务可复用的 registry |
| 4 | **翻译** | 按句 / 单元对齐翻译 | local-m2m100（离线）/ deepseek（在线） |
| 5 | **合成** | 用单说话人音色合成配音 | moss-tts-nano-onnx（默认）/ qwen3tts / voxcpm2 |
| 6 | **回贴** | 把配音对齐回原时间轴 | atempo / rubberband 变速 + 侧链混音 |
| 7 | **交付** | 混音、封装、可选烧字幕 | FFmpeg mux |

> 💡 同一段阶段代码有两个入口：跑完整流水线时由编排器调用，单独用「原子工具」时直接调用。所以你在[原子工具](/guide/atomic-tools)里看到的「分离 / 转写 / 翻译…」就是这些阶段本身。

## 模板：决定跑哪些阶段

你不必逐个勾选阶段——选一个**工作流模板**，它会点亮对应的阶段子图：

| 模板 | 包含什么 |
| --- | --- |
| `asr-dub-basic` | 基础配音链路（仅七个主干阶段）。**默认模板**。 |
| `asr-dub+visual` | 在基础上加「画面感知」（本地 Qwen3-VL）：按场景描述注入翻译上下文，减少称谓 / 指代 / 语气误译。 |
| `asr-dub+ocr-subs` | 在基础上加 OCR 字幕识别 / 翻译，并用 OCR 结果校正听写文本。 |
| `asr-dub+ocr-subs+erase` | 在上一项基础上再加原片硬字幕擦除。 |

新建任务时，你通常**不用直接选模板**——选好「成品目标」与「质量档位」后，向导会替你挑合适的模板（高级用户可在「开发者设置」里手动覆盖）。详见[完整配音流水线](/guide/pipeline-dubbing)。

## 在任务详情里看阶段

任务跑起来后，详情页用一张实时 **DAG 图**展示每个阶段的状态与进度：

![任务详情页的阶段 DAG 与进度](/guide/pipeline-graph.png)

*图 2：每个节点是一个阶段，颜色表示状态；点节点可展开该阶段的日志与产物。*

阶段状态一览：

| 状态 | 含义 |
| --- | --- |
| **等待中** | 还没轮到，前置阶段未完成。 |
| **运行中** | 正在执行，节点上显示进度条。 |
| **已完成** | 成功产出全部产物。 |
| **命中缓存** | 输入没变，直接复用上次结果（见下文）。 |
| **失败** | 出错并中止，可在节点里看错误与日志。 |
| **跳过** | 当前模板 / 设置下不需要此阶段。 |

整条任务则有「成功 / 部分成功 / 失败」之分——**部分成功**意味着主链路出了成片，但某些可选支线（如字幕擦除）没完全跑通，仍可交付。

## 缓存：为什么重跑这么快

每个阶段都会算一个**缓存键**：把该阶段的参数 + 上游产物的指纹哈希在一起。只有当 manifest 存在、状态为成功、产物文件都在、且缓存键没变时，这个阶段才算**命中缓存**，直接跳过。

这带来两个你能直接感知的行为：

- **改一个设置只重算受影响的部分。** 比如你只换了**翻译后端**，那么分离、转写、说话人注册会命中缓存秒过，只有翻译及其下游（合成 → 回贴 → 交付）重算。
- **可以从任意阶段重跑。** 在任务详情页选一个起始阶段「从所选阶段重跑」，它之前的阶段照样复用缓存。

![任务详情页全貌：进度、阶段图、交付与产物](/guide/task-detail.png)

*图 3：任务详情页把进度、阶段图、说话人核对、交付与成品素材清单集中在一页。*

> 💡 想强制重算？改动相关设置（缓存键会变），或在新建任务时关掉「使用缓存」。删除任务会一并清掉它的产物目录。
