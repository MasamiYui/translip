# 配音评测与一键修复

「配音评测」对一条已完成配音做**逐段质检**：打一个总分、定一个交付结论、列出每一段的具体问题，并把这些问题转成一份可执行的修复计划——能一键自动修复，也能导出给人工跟进。

入口：左侧「**配音评测**」。列表里是可评测的任务，可按状态筛选、按名称搜索。

![配音评测列表](/guide/evaluation-list.png)

*图 1：评测列表——每条任务显示总分与问题数，点进去看详情。*

## 跑一次评测

点进某条任务，点「**开始评测**」生成报告（已有报告时是「重新评测」）。还没跑过时是这样的空状态：

![配音评测的空状态与入口](/guide/evaluation-detail.png)

*图 2：勾选「启用翻译质量打分」可额外用 LLM 评估翻译质量（需 DeepSeek Key）。*

> 💡 「启用翻译质量打分」会调用 DeepSeek 对译文质量单独打分，需要在[全局设置](/guide/settings)里配好 `DEEPSEEK_API_KEY`；不勾也能跑其余所有指标。

## 看懂评分卡

报告头部是一张**评分卡**：

![配音评测详情：评分卡、交付门与修复计划](/guide/evaluation-detail-full.png)

*图 3：总分、配音覆盖率、问题片段数、翻译打分状态，以及一排「交付门」。*

| 指标 | 含义 |
| --- | --- |
| **综合得分** | 0–100 的总分，颜色分档（高=绿、中=黄、低=红）。 |
| **交付结论** | `可交付候选` / `需复核` / `阻断`——一句话告诉你能不能发。 |
| **配音覆盖率** | 已配音段 / 应配音段，漏配会拉低它。 |
| **问题片段** | 被判定有问题的段数。 |
| **交付门** | 几道硬性门槛（如字幕可读覆盖率、说话人一致性、音色风险、漏配片段），逐项给出通过 / 未通过。 |

## 逐段问题

往下是**片段列表**，可只看「有问题」的段，或按问题类型筛选。点任意一段打开抽屉，里面有：原声 vs 配音的 A/B 试听、问题诊断（根因 + 建议动作）、原文 / 译文 / 回译对照，以及说话人相似度、文本相似度、时长比等指标。

常见的问题标签：

| 标签 | 含义 |
| --- | --- |
| **漏配** | 该说的话没配上（最影响覆盖率）。 |
| **节奏** | 配音过快 / 过慢，没贴合原片窗口。 |
| **掉字 / 不可懂** | 合成吞字或听不清。 |
| **音色不匹配** | 配音音色与该说话人参考差异过大。 |
| **翻译问题** | 译文有误或不自然（启用翻译打分时更准）。 |

## 一键修复

评分卡下方的「**下一步优化**」把所有问题归并成一份**按优先级排好的修复计划**，并标出哪些可以自动修：

| 修复动作 | 说明 |
| --- | --- |
| **更换音色参考** | 给音色不匹配的说话人换更合适的参考。 |
| **重新合成** | 对掉字 / 节奏差的段重跑 TTS。 |
| **缩短 / 改写译文** | 译文太长贴不进窗口时压缩改写。 |
| **重新翻译** | 翻译有误时重译该段。 |
| **补配漏配段** | 把漏掉的段补上。 |

你有三种用法：

- **一键自动修复**——让系统按计划自动重合成 / 重渲染并再评一遍，循环逼近达标。
- **定位片段**——跳到片段列表里对应的段，手动去[配音编辑台](/guide/dubbing-editor)精修。
- **导出优化指令**——把计划导成 JSON，交给别的流程或人工处理。

> 💡 评测 → 修复 → 再评测是一个闭环。达到「可交付候选」后，回[详情页交付](/guide/pipeline-dubbing)导出成片即可。
