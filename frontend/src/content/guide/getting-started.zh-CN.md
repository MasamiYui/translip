# 快速开始

**translip** 是一个本地优先、多说话人感知的视频配音流水线：把「音频分离 → 说话人归因转写 → 翻译 → 单说话人 TTS → 时间轴回贴与混音 → 视频交付」串成一条可复用的端到端流程。它既能一键跑完整条链路，也能把每个环节当作独立的「原子工具」单独使用，还配有一个完整的 Web 管理界面。

> 💡 这份文档面向**使用者**——只讲怎么用界面把活干完。需要 CLI 参数、源码结构或部署细节，请看仓库根目录的 `README.md`。

## 它能帮你做什么

- **多说话人配音**：自动分离人声与背景，区分不同说话人，逐句翻译并用单说话人音色合成配音，再贴回原时间轴。
- **保留背景音**：人声与背景分轨处理，配音只替换人声，环境音、音乐、音效照常保留。
- **硬字幕识别与擦除**：可选地识别原片烧录字幕、用它校正听写文本，甚至把原字幕擦掉重建干净底板。
- **可视化校对**：内置「配音编辑台」与「配音评测」，逐段试听、改词、重合成、打分、一键修复。
- **原子工具 + AI 助手**：任意单个环节都能单独跑；说不清步骤时，让 AI 助手用一句话帮你编排。

## 三种使用方式

同一套能力有三个入口，按你的目标选：

| 入口 | 适合 | 在文档里看 |
| --- | --- | --- |
| **完整配音流水线** | 想要一条可直接发布的成片 | [完整配音流水线](/guide/pipeline-dubbing) |
| **原子工具** | 只想要某一步的产物（如只做分离、只做字幕） | [原子工具](/guide/atomic-tools) |
| **AI 助手** | 说不清具体步骤，想让系统替你规划 | [AI 助手](/guide/ai-assistant) |

## 准备环境

translip 在本地运行，启动前确认这些就绪：

| 依赖 | 说明 |
| --- | --- |
| **FFmpeg** | 必须在 `PATH` 上——分离、混音、合并、字幕烧录全靠它。 |
| **uv** | Python 环境与依赖管理（不是 pip / poetry）。`uv sync` 安装运行依赖。 |
| **Node + npm** | 仅在需要自行构建前端时用到（`npm install && npm run build`）。 |

> ⚠️ 部分能力依赖额外组件：默认 TTS 后端需要单独安装 `moss-tts-nano` CLI；字幕识别需要 `ocr` 额外依赖；`deepseek` 翻译与翻译质量评分需要 `DEEPSEEK_API_KEY`。缺了会在用到时给出明确报错，不影响其余功能。详见[常见问题](/guide/faq)。

## 启动管理界面

最简单的一次性安装方式——构建好前端，让后端把界面和 API 一起托管：

```bash
uv sync                 # 安装后端依赖（FFmpeg 需已在 PATH 上）
cd frontend && npm install && npm run build && cd ..
uv run translip-server  # 启动控制面，自动挂载 frontend/dist
```

启动后打开 **http://127.0.0.1:8765** 即可。需要边改边调的开发模式，可以另跑 `npm run dev`（Vite 开发服务器在 `:5173`，自动把 `/api` 代理到 `:8765`）。

## 5 分钟跑通第一条配音

1. 点右上角「**新建任务**」（或左侧「任务中心 → 新建流水线任务」）。
2. 选一段视频、设定**源语言 / 目标语言**，其余保持默认。
3. 一路「下一步」到最后，点「**创建任务**」。
4. 在任务详情页看**阶段 DAG** 实时跑完，到 100%。
5. 在底部「交付」处导出成片，或先进「配音编辑台」精修。

完整、逐步、带截图的走法，见下一章 [完整配音流水线](/guide/pipeline-dubbing)。想先理解流水线为什么这么设计，读 [核心概念](/guide/concepts)。
