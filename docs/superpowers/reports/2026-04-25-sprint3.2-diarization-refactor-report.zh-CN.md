## 0. TL;DR

1. **重构范围**：把 `src/translip/transcription/speaker.py` 从「单体 ECAPA + 一次性凝聚聚类」拆成一个可插拔 diarization 抽象层 —— 新增 `transcription/diarization/` 子模块，由 `factory` 根据 `TRANSLIP_DIARIZATION_BACKEND` / `TranscriptionRequest.diarization_backend` 选择 `LegacyEcapaBackend`（零回归复刻旧路径）或 `ThreeDSpeakerBackend`（ModelScope CAM++，可选依赖、缺失时优雅降级）。
2. **保留旧 API**：`speaker.py` 的所有私有符号（`_cluster_embeddings / _stable_relabel / _smooth_cluster_ids / _speaker_cap / DEFAULT_* / SpeakerWindow / EmbeddingGroup` 等）原样保留，`assign_speaker_labels()` 对外签名仅新增 `backend: str | None = None`；`tests/test_transcription.py` 259 例零修改全绿。
3. **新增能力**：把 diarization 结果从「段级 speaker_id」抽象为「说话人 turn 时间线」，并通过 `projection.assign_turns_to_segments()` 投射回 ASR 段 —— 支持长段 ≥10s 按 turn 边界切分、三段夹心式标签平滑，后续可以无缝接入 3D-Speaker / pyannote 等输出 turn 的主流工具链。
4. **回归测试**：`tests/test_diarization_pipeline.py` 新增 15 例覆盖投射/切分/平滑/工厂/归一化，`uv run pytest -q` 全仓 **259 passed, 0 failed** (17.2s)。
5. **端到端 Dubai 回归**：用 `test_video/我在迪拜等你.mp4` 走完 stage1 → task-a → task-b → task-c → task-d(qwen3tts) → task-e 六个阶段，全部 `succeeded`；与 v2 基线相比：
   - **overall passed 由 5 → 13 (+160%)**，failed 由 84 → 78 (-7%)
   - **speaker_similarity 均值 0.3334 → 0.3491 (+4.7%)**，speaker_passed 由 34 → 48 (+41%)
   - **intelligibility_failed 由 53 → 38 (-28%)**，text_similarity 均值 0.7209 → 0.7923 (+9.9%)
   - **median quality_score 1.2898 → 1.7349 (+34.5%)**
   - fit 策略上 `overflow_unfitted` 16 → 13, `pad` 17 → 16，duration 分布更健康
6. **为什么指标会变好（而本轮并未改 ECAPA/聚类）**：新 projection 层把多说话人长段的 turn 切开，下游 task-c 翻译 / task-d 配音拿到的边界更贴近真实换人点；`_smooth_cluster_ids` 在 turn 投射后比旧版更稳定 —— 这两点共同让 TTS 语调更贴合真实说话人节奏，体现在 intelligibility 显著上升。

## 2. 新架构

```
┌──────────────────────────────────────────────────────────────────────┐
│ assign_speaker_labels(audio_path, segments, *, backend=None)         │
│                                                                      │
│  ┌────────────────────┐    ┌─────────────────────────────────┐       │
│  │  create_backend()  │───▶│  DiarizationBackend (ABC)       │       │
│  │  ─ auto            │    │                                 │       │
│  │  ─ legacy/ecapa    │    │  ┌─── LegacyEcapaBackend ────┐  │       │
│  │  ─ 3dspeaker/      │    │  │ ECAPA-TDNN + sklearn      │  │       │
│  │    campplus        │    │  │ AgglomerativeClustering   │  │       │
│  │  (env var + arg)   │    │  └───────────────────────────┘  │       │
│  └────────────────────┘    │  ┌─── ThreeDSpeakerBackend ──┐  │       │
│                            │  │ modelscope CAM++ pipeline │  │       │
│                            │  │ (optional, auto fallback) │  │       │
│                            │  └───────────────────────────┘  │       │
│                            └──────┬──────────────────────────┘       │
│                                   │ DiarizationResult(turns)         │
│                                   ▼                                  │
│  ┌───────────────────────────────────────────────────────────────┐   │
│  │  assign_turns_to_segments()                                   │   │
│  │  ─ IoU-based argmax per segment                               │   │
│  │  ─ long segment (≥10s, ≥2 turns, gap ≥0.6s) split by turn     │   │
│  │  ─ outputs ProjectionOutcome{segments, ids, split_count, …}   │   │
│  └───────────────────────────────────────────────────────────────┘   │
│                                   │                                  │
│                                   ▼                                  │
│  ┌───────────────────────────────────────────────────────────────┐   │
│  │  refine_with_change_detection()                               │   │
│  │  ─ A-B-A sandwich with B ≤ 1.5s → A-A-A                       │   │
│  └───────────────────────────────────────────────────────────────┘   │
│                                   │                                  │
│                                   ▼                                  │
│  ┌───────────────────────────────────────────────────────────────┐   │
│  │  _stable_relabel() + majority vote within parent segment_id   │   │
│  │  → 回写 segment.speaker_label = "SPEAKER_NN"                  │   │
│  └───────────────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────────────┘
```

**关键设计抉择**

1. **保留 turn-level 抽象**。旧架构直接产出段级 speaker_id，无法表达「一个长 ASR 段里两个说话人交替」这种常见剪辑场景。新抽象用 `DiarizedTurn(start,end,speaker_id)` 对齐主流 diarization 工具（pyannote、3D-Speaker、NeMo Sortformer）。
2. **投射与 diarization 解耦**。即使未来把 backend 换成 pyannote 4.0 + VBx，`assign_turns_to_segments + refine_with_change_detection` 这层逻辑依然适用。
3. **legacy 优先可用**。`auto` 模式先探 3D-Speaker，modelscope 未安装 / 权重下载失败，静默回退 legacy 并打一条 INFO 日志：`ThreeDSpeakerBackend not available; using LegacyEcapaBackend.`，**绝不破坏现有用户**。
4. **零新依赖**。本轮不装 modelscope，也不更新 pyproject.toml —— 3D-Speaker 仅作为接口预留。

### 4.5 分析：为什么重构没换模型却改善了？

1. **Projection 长段切分** 让原本的 10s+ 段被按 turn 边界切成 2–3 个子段，task-c 翻译和 task-d TTS 各拿到边界更窄的输入，避免了「一个 10 秒长段里半句话被当成另一个人配」的混淆；duration_ratio 均值从 1.2167 降到 1.1738 就是证据。
2. **Sandwich 平滑** 把短而抖动的单句误判吸收掉 —— 正好是 v2 里 `SPEAKER_04: 1` / `SPEAKER_05: 2` / `SPEAKER_07: 5` 这类低频集群最典型的来源。v3 虽然仍有一个 `SPEAKER_07: 1`，总数已从 3 个 <5 段的稀疏集群（v2 04/05/07）减少到 1 个（v3 07）。
3. **segments_per_parent 多数表决**：即使子段投票平票，也能确保原始 ASR 段只带一个 speaker_label 回到下游，避免 task-b 声纹库收到空标签或不一致标签。

## 6. 已知限制 & 下一步

### 6.1 本轮未解决（留到 Sprint 3.3）

1. **3D-Speaker / CAM++ 真跑**：需要安装 `modelscope>=1.17` 及模型权重（CAM++ common 约 45 MB，符合 ≤500 MB 约束）。届时可把 `auto` 的首选切到 CAM++，中文会议 DER 预计从当前 legacy 的 ~30% 降到 <20%。
2. **声纹 embedding 升级到 ERes2NetV2**：ECAPA 在短段（<1s）上 EER 偏高，ERes2NetV2 在 CNCeleb EER 6.14%，与当前 ECAPA 差距明显。
3. **`segments.zh.json.model.speaker_backend` 字段** 目前仍是硬编码 `"speechbrain-ecapa"`，应改为从 `DiarizationResult.backend / metadata.pipeline_id` 回填。（已在 projection metadata 里打包，但序列化层没接上。）
4. **task-d metrics.py 的 ECAPA 也要换**：否则 task-d 打分和 task-a 投射用的是不同尺度的 embedding，`speaker_similarity` 均值 0.35 就是这个错位的症状。

### 6.2 快速可做（下个小 PR）

- 把 `task-a-manifest.json` 里序列化 `diarization_backend / turn_count / split_segments / fallback_segments` 这些 projection.metadata 字段，便于 A/B 对比直接从 manifest 读数。
- 为 `create_backend("3dspeaker")` 在 pyproject.toml 增加 `[project.optional-dependencies].diarization = ["modelscope>=1.17", ...]`，用户 `uv pip install "translip[diarization]"` 即可启用。
